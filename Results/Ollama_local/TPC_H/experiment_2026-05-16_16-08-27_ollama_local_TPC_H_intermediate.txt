## 1. Workload-Driven Analysis

Based on the provided schema (standard TPC-H structure), this is a classic **OLTP/OLAP hybrid** design optimized for writes but inefficient for reads.

**Identified Bottlenecks:**
1.  **The "Join Hell" Problem:** Queries requiring Part/Supplier details must join `LINEITEM` $\rightarrow$ `PART` $\rightarrow$ `SUPPLIER` $\rightarrow$ `NATION` $\rightarrow$ `REGION`.
2.  **Data Locality:** `NATION` and `REGION` are tiny tables. Joining them adds overhead for every query.
3.  **Filtering Overhead:** Filtering by `OrderStatus` or `ShipDate` requires scanning the full `LINEITEM` table, then joining to `ORDERS` to filter.
4.  **Aggregation Latency:** Reporting on "Total Spend per Customer" requires a complex join chain and aggregation every time.

**Goal:** Create a **Read-Optimized Schema** (Materialized/Denormalized) that eliminates these joins at the cost of storage and consistency (handled via ETL/Triggers).

---

## 2. Denormalization Strategy

We will adopt a **"Wide Table"** strategy for the read layer.
*   **Dimensions:** Merge `REGION` into `NATION`. Merge `NATION` into `CUSTOMER` and `SUPPLIER`.
*   **Facts:** Merge `PART` and `SUPPLIER` details into `LINEITEM`.
*   **Aggregates:** Pre-calculate daily/monthly sales summaries.
*   **Partitions:** Partition large tables by `ShipDate` (or `OrderDate`) to enable efficient pruning.

---

## 3. New Physical Schema (DDL)

This DDL assumes a PostgreSQL/MySQL compatible syntax.
*   **Note:** `FACT_` tables are designed to be **Read-Only** or updated asynchronously (via ETL/Triggers).
*   **Note:** `DIM_` tables are the "Wide" versions containing all necessary context.

### A. Denormalized Dimensions (Wide Tables)

```sql
-- 1. DIM_NATION_REGION (Merged NATION + REGION)
-- Purpose: Eliminates N + R joins. Stores RegionName directly.
CREATE TABLE DIM_NATION_REGION (
    NATION_KEY        INT PRIMARY KEY,
    NATION_NAME       VARCHAR(25) NOT NULL,
    REGION_NAME       VARCHAR(25) NOT NULL, -- Merged from REGION table
    NATION_REGION_KEY BIGINT GENERATED ALWAYS AS (NATION_KEY * 1000 + HASH(REGION_NAME)) STORED, -- Optional composite key
    UPDATED_AT        TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```
