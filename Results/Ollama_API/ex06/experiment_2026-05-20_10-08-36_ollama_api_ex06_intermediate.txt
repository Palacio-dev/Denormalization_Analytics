## 🚀 Step 2: Denormalization Design

We’ll apply several strategies:

| Strategy | Use Case |
|---------|----------|
| **Pre-Joined Wide Tables** | For frequently joined entities like School-Municipality-State |
| **Redundant Attributes** | Add `nome_municipio`, `sigla_uf` directly into `ESCOLA`, `IES`, etc. |
| **Aggregate Tables** | Summarize per-year or per-institution stats (e.g., avg Enade per university/year) |
| **Materialized Views / Covering Indexes** | To speed up complex aggregations |

### ⚙️ Table-Level Changes

#### ➤ 1. `ESCOLA`: Include Municipality & State Names
Denormalize `MUNICIPIO.nome_municipio` and `UF.sigla_uf` into this table to avoid two joins.

#### ➤ 2. `IES`: Include Municipality & State Info
Same rationale as above — store `nome_municipio`, `sigla_uf` redundantly.

#### ➤ 3. Create Materialized View: `VW_CURSO_ENADE_YEARLY`
For efficient retrieval of course-level Enade results with associated institution/municipality/state info.

#### ➤ 4. Aggregate Table: `AGG_ENADE_PER_IES_YEAR`
Store pre-aggregated yearly Enade scores at the university level to eliminate real-time joins/groupings.

#### ➤ 5. Add Foreign Key from `MUNICIPIO` to `ANO`
Fix schema so municipalities can have multiple years of educational indicators via intermediate table.

## 📐 Final Denormalized Physical Schema

Below is the optimized **physical schema DDL** incorporating all changes:

```sql
-- Base tables remain mostly unchanged except minor additions
DROP TABLE IF EXISTS UF CASCADE;
CREATE TABLE UF (
    id INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    sigla_uf VARCHAR(2) UNIQUE NOT NULL,
    nome VARCHAR(100) NOT NULL,
    qtd_habitantes INT CHECK(qtd_habitantes >= 0)
);

DROP TABLE IF EXISTS MUNICIPIO CASCADE;
CREATE TABLE MUNICIPIO (
    id_municipio INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_uf INT NOT NULL REFERENCES UF(id) ON DELETE CASCADE,
    nome_municipio VARCHAR(100) NOT NULL,
    qtd_IES INT CHECK (qtd_IES >= 0),
    qtd_ESCOLAS INT CHECK (qtd_ESCOLAS >= 0),
    media_ideb REAL,
    media_saeb REAL,
    media_enade REAL,
    total_participantes INT CHECK (total_participantes >= 0)
);

-- Keep other dimension tables as-is
-- IDEB, ENADE, SAEB, IES, ESCOLA, CURSO, ANO definitions unchanged until modified below...

-- Modified ESCOLA table with denormalized fields
DROP TABLE IF EXISTS ESCOLA CASCADE;
CREATE TABLE ESCOLA (
    id_escola INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_municipio INT NOT NULL REFERENCES MUNICIPIO(id_municipio) ON DELETE CASCADE,
    nome_escola VARCHAR(100) NOT NULL,
    cod_escola INTEGER UNIQUE NOT NULL,
    rede VARCHAR(100) NOT NULL,
    nome_municipio VARCHAR(100) NOT NULL,  -- Redundant field
    sigla_uf VARCHAR(2) NOT NULL          -- Redundant field
);

-- Modified IES table with denormalized fields
DROP TABLE IF EXISTS IES CASCADE;
CREATE TABLE IES (
    id_ies INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    id_municipio INT NOT NULL REFERENCES MUNICIPIO(id_municipio) ON DELETE CASCADE,
    nome_ies VARCHAR(100) NOT NULL,
    sigla_ies VARCHAR(20),
    codigo_ies INT UNIQUE NOT NULL,
    rede VARCHAR(100) NOT NULL,
    nome_municipio VARCHAR(100) NOT NULL,  -- Redundant field
    sigla_uf VARCHAR(2) NOT NULL           -- Redundant field
);

-- Aggregate table for fast access to IES' annual Enade averages
DROP TABLE IF EXISTS AGG_ENADE_PER_IES_YEAR CASCADE;
CREATE TABLE AGG_ENADE_PER_IES_YEAR (
    id_ies INT NOT NULL,
    ano INT NOT NULL,
    avg_nota_continua REAL,
    count_cursos INT,
    PRIMARY KEY (id_ies, ano)
);

-- Materialized view equivalent using a base table for performance
DROP TABLE IF EXISTS VW_CURSO_ENADE_YEARLY CASCADE;
CREATE TABLE VW_CURSO_ENADE_YEARLY (
    id_curso INT,
    nome_curso VARCHAR(100),
    id_ies INT,
    nome_ies VARCHAR(100),
    sigla_ies VARCHAR(20),
    ano INT,
    nota_enade_continua REAL,
    nota_padronizada_ce REAL,
    nota_padronizada_fg REAL,
    id_municipio INT,
    nome_municipio VARCHAR(100),
    sigla_uf VARCHAR(2),
    PRIMARY KEY (id_curso, ano)
);
```
