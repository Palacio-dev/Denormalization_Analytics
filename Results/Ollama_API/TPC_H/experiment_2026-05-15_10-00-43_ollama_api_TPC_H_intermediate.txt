## ✅ 1. Workload-Driven Analysis

### Common Read Queries (Inferred from TPC-H-like schema):
1. **Customer Orders with Lineitem Details**
   - Join: CUSTOMER → ORDERS → LINEITEM → PART
   - Bottleneck: Multiple joins, especially CUSTOMER→ORDERS→LINEITEM

2. **Supplier Performance Analysis**
   - Join: SUPPLIER → PARTSUPP → LINEITEM
   - Bottleneck: PARTSUPP → LINEITEM join on PS_PARTKEY = L_PARTKEY and PS_SUPPKEY = L_SUPPKEY

3. **Order Profitability / Revenue by Date**
   - Join: ORDERS → LINEITEM
   - Aggregate: SUM(L_EXTENDEDPRICE * (1 - L_DISCOUNT))

4. **Nation/Region-based Aggregations**
   - CUSTOMER → NATION → REGION
   - SUPPLIER → NATION → REGION

### Denormalization Candidates:
- **CUSTOMER + ORDERS + LINEITEM** → Wide "ORDER_LINEITEM_CUSTOMER" table
- **SUPPLIER + PARTSUPP + LINEITEM** → Wide "SUPPLY_LINEITEM" table
- **ORDERS + LINEITEM aggregates** → Summary table by date/order
- **Nation/Region fields** → Embedded in CUSTOMER and SUPPLIER
- **Part Name** → Embedded in LINEITEM

## ✅ 2. Denormalization Design

  - N_NAME (Nation), R_NAME (Region)

#### Table: `SUPPLY_LINEITEM`
- **Purpose**: Eliminate joins between SUPPLIER, PARTSUPP, LINEITEM
- **Grain**: One row per lineitem, enriched with supplier/part metadata
- **Includes**:
  - L_ORDERKEY, L_PARTKEY, L_SUPPKEY, L_QUANTITY, L_EXTENDEDPRICE, L_DISCOUNT
  - S_NAME, S_NATIONKEY
  - PS_AVAILQTY, PS_SUPPLYCOST
  - N_NAME (Nation), R_NAME (Region)

## ✅ 3. New Physical Schema (DDL)

```sql
-- Wide Table: Order + Lineitem + Customer
CREATE TABLE ORDER_LINEITEM_CUSTOMER (
    L_ORDERKEY       INTEGER NOT NULL,
    L_LINENUMBER     INTEGER NOT NULL,
    L_QUANTITY       DECIMAL(15,2) NOT NULL,
    L_EXTENDEDPRICE  DECIMAL(15,2) NOT NULL,
    L_DISCOUNT       DECIMAL(15,2) NOT NULL,
    L_TAX            DECIMAL(15,2) NOT NULL,
    L_SHIPDATE       DATE NOT NULL,

    O_ORDERDATE      DATE NOT NULL,
    O_ORDERSTATUS    CHAR(1) NOT NULL,
    O_TOTALPRICE     DECIMAL(15,2) NOT NULL,

    C_CUSTKEY        INTEGER NOT NULL,
    C_NAME           VARCHAR(25) NOT NULL,
    C_NATIONKEY      INTEGER NOT NULL,
    C_MKTSEGMENT     CHAR(10) NOT NULL,

    N_NAME           CHAR(25) NOT NULL,  -- Denormalized Nation Name
    R_NAME           CHAR(25) NOT NULL   -- Denormalized Region Name
);

CREATE INDEX idx_olc_shipdate_nation_segment ON ORDER_LINEITEM_CUSTOMER (L_SHIPDATE, C_NATIONKEY, C_MKTSEGMENT);
CREATE INDEX idx_olc_cust_orderdate ON ORDER_LINEITEM_CUSTOMER (C_CUSTKEY, O_ORDERDATE);

-- Wide Table: Supplier + PartSupp + Lineitem
CREATE TABLE SUPPLY_LINEITEM (
    L_ORDERKEY       INTEGER NOT NULL,
    L_PARTKEY        INTEGER NOT NULL,
    L_SUPPKEY        INTEGER NOT NULL,
    L_QUANTITY       DECIMAL(15,2) NOT NULL,
    L_EXTENDEDPRICE  DECIMAL(15,2) NOT NULL,
    L_DISCOUNT       DECIMAL(15,2) NOT NULL,
    L_SHIPDATE       DATE NOT NULL,

    S_NAME           CHAR(25) NOT NULL,
    S_NATIONKEY      INTEGER NOT NULL,

    PS_AVAILQTY      INTEGER NOT NULL,
    PS_SUPPLYCOST    DECIMAL(15,2) NOT NULL,

    N_NAME           CHAR(25) NOT NULL,  -- Denormalized Nation Name
    R_NAME           CHAR(25) NOT NULL   -- Denormalized Region Name
);

CREATE INDEX idx_sl_supp_part ON SUPPLY_LINEITEM (L_SUPPKEY, L_PARTKEY);
CREATE INDEX idx_sl_shipdate_nation ON SUPPLY_LINEITEM (L_SHIPDATE, S_NATIONKEY);

-- Summary Table: Daily Order Summary
CREATE TABLE DAILY_ORDER_SUMMARY (
    ORDERDATE        DATE NOT NULL,
    TOTAL_REVENUE    DECIMAL(18,2),
    TOTAL_ORDERS     INTEGER,
    TOTAL_ITEMS      INTEGER,
    AVG_DISCOUNT     DECIMAL(5,2)
);

CREATE INDEX idx_dos_orderdate ON DAILY_ORDER_SUMMARY (ORDERDATE);

-- Summary Table: Customer Order Summary
CREATE TABLE CUSTOMER_ORDER_SUMMARY (
    C_CUSTKEY           INTEGER NOT NULL,
    TOTAL_REVENUE       DECIMAL(18,2),
    TOTAL_ORDERS        INTEGER,
    LAST_ORDER_DATE     DATE
);

CREATE INDEX idx_cos_custkey ON CUSTOMER_ORDER_SUMMARY (C_CUSTKEY);

-- Optional: Denormalized LINEITEM with Part Name
-- Only if Part Name is frequently accessed with Lineitem
ALTER TABLE LINEITEM ADD COLUMN P_NAME VARCHAR(55);
-- (You would populate this from the PART table during ETL)

-- Existing Index Enhancements
CREATE INDEX idx_lineitem_shipdate ON LINEITEM (L_SHIPDATE);
CREATE INDEX idx_lineitem_order_line ON LINEITEM (L_ORDERKEY, L_LINENUMBER);
```
