1. Creating a Death Table

Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.
The ideia as to merge the information from the Mother table, Deceased table , the Death table and the Death cause. All of 
these tables have information about death, so it makes sense to put them together. Besides, collapse relations is user to
collapse the one to many relationship between death and death cause.

2. Creating a Consolidated "Location" Dimension

The normalized model separated geographic hierarchy (State → Municipality) and infrastructure (BasicHealthUnit).
The denormalized model creates a single Location table.

    Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.

    The Idea: The Location table serves as a "Wide Table" or "Flat Dimension." It combines city data, state names,
    and health unit details into one row per city/unit.


It is important to notice that the attibutes "IBGE_city_code" and "location_code" are connected , this way, it is
possible to get all the information previously possible on the normalized model.
