# Denormalized Read-Optimized Physical Schema Design

## Executive Summary

This document presents a comprehensive denormalization strategy to optimize read performance for the given normalized schema while preserving business correctness. The approach focuses on collapsing join-heavy queries into wide read tables that pre-join frequently accessed dimension data. The design maintains data freshness through batch refresh cycles during limited maintenance windows, with explicit handling of operational constraints including no trigger support. All denormalized artifacts are tied directly to specific query patterns identified from the workload analysis.

## Proposed Denormalized Schema (DDL)

```sql
-- Main denormalized read table optimized for common query patterns
CREATE TABLE IF NOT EXISTS public."UsinaReadOptimized" (
    -- Primary key from source table
    id INTEGER PRIMARY KEY,
    
    -- Core usina attributes
    nome VARCHAR NOT NULL,
    potkw FLOAT,
    potfis FLOAT,
    iniciop DATE,
    atualizacao DATE,
    lat DOUBLE PRECISION,
    long DOUBLE PRECISION,
    estagio VARCHAR,
    
    -- Denormalized dimension data
    proprietario VARCHAR,  -- From Proprietarios table
    rio VARCHAR NOT NULL,  -- From Rios table
    
    -- Location hierarchy flattened
    id_municipio1 INTEGER NOT NULL,  -- Reference to Municipios
    munic1 VARCHAR NOT NULL,          -- Municipio name
    uf1 CHAR(2) NOT NULL,            -- Estado code
    
    id_municipio2 INTEGER,           -- Optional second municipio
    munic2 VARCHAR,                  -- Second municipio name
    uf2 CHAR(2),                     -- Second estado code
    
    -- Metadata for consistency tracking
    _last_refreshed TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    _source_hash TEXT  -- For detecting changes in source data
);

-- Indexes optimized for common query patterns
CREATE INDEX idx_usina_read_potkw ON public."UsinaReadOptimized" (potkw DESC);
CREATE INDEX idx_usina_read_atualizacao ON public."UsinaReadOptimized" (atualizacao DESC);
CREATE INDEX idx_usina_read_uf1 ON public."UsinaReadOptimized" (uf1);
CREATE INDEX idx_usina_read_rio ON public."UsinaReadOptimized" (rio);
CREATE INDEX idx_usina_read_proprietario ON public."UsinaReadOptimized" (proprietario) WHERE proprietario IS NOT NULL;

-- Optional: Partition by estado for large datasets
```
