1. Creating a dim_escola_indicadores_anuais table

Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.

The attibutes from the following table were united in a single table: escola,desempenho_enem, localizacao, indicadores_escolares,
participacao_enem.This way, the one-to-may relationships are collapsed, and redundant attributes
that would just get searched from a JOIN query are now on the same big table.

2. Creating a fato_aluno_trajetoria table

Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.

The attibutes from the following table were united in a single table: curso_ifb, matricula_ifb, aluno_ifb, vinculo_empregaticio
and remuneracao.This way, the one-to-may relationships are collapsed, and redundant attributes
that would just get searched from a JOIN query are now on the same big table.


It is important to notice that the tables are connected with the "unidade_ensino" attribute and the "no_escola_educacenso"
attibute, which is a relationship one-to-one , basically, one student has studied in one "unidade_ensino" which is the
same name as "no_escola_educacenso". This way,it is possible to get all the information previously possible on the
normalized model.
