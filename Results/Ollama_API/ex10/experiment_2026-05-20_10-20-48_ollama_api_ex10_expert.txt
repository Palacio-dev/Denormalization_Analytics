# Proposed Denormalized Schema (DDL)

We will create two main read-optimized artifacts:
1. A **wide fact table** combining emissions, population, and energy usage per country/year.
2. A **dimensional snapshot** of current countries with their continents.

These artifacts are refreshed daily via batch jobs using CDC or full-table scans from source tables.

## Artifact 1: Wide Fact Table (`fato_energia_emissao_populacao`)

**Purpose**: Pre-joined view of emissions, energy consumption, and population grouped by country and year.

```sql
CREATE TABLE IF NOT EXISTS public.fato_energia_emissao_populacao (
    id_paises INTEGER NOT NULL,
    nome_pais VARCHAR(50),
    ano INTEGER NOT NULL,
    continente_nome VARCHAR(50),
    total_quantidade_emitida BIGINT,
    total_porcentagem_consumo INTEGER,
    total_populacao INTEGER,

CREATE INDEX idx_fato_ano ON public.fato_energia_emissao_populacao (ano);
```

# Implementation Plan

## Phase 1: Preparation
- Create target denormalized tables.
- Add monitoring for:
  - Lag between source updates and target refreshes.

## Phase 2: Batch Job Deployment
- Daily ETL job scheduled (e.g., cron or Airflow):
  - Truncate and rebuild `dim_pais_continente`.
  - Aggregate and populate `fato_energia_emissao_populacao`.

Example pseudo logic:
```sql
-- Refresh dim_pais_continente
INSERT INTO dim_pais_continente (id_paises, nome_pais, nome_continente)
SELECT p.id_paises, p.nome, c.nome
FROM paises p
JOIN continente c ON p.continente = c.id_continente;

-- Refresh fato_energia_emissao_populacao
INSERT INTO fato_energia_emissao_populacao (
    id_paises, nome_pais, ano, continente_nome,
    total_quantidade_emitida, total_porcentagem_consumo, total_populacao
)
SELECT 
    p.id_paises,
    p.nome AS nome_pais,
    COALESCE(e.ano, c.ano, pop.ano) AS ano,
    cont.nome AS continente_nome,
    SUM(e.quantidade_emitida) AS total_quantidade_emitida,
    SUM(c.porcentagem) AS total_porcentagem_consumo,
    SUM(pop.quantidade_hab) AS total_populacao
FROM paises p
LEFT JOIN continente cont ON p.continente = cont.id_continente
LEFT JOIN emissao_g_efeito_estufa e ON p.id_paises = e.id_paises
LEFT JOIN consumo_anual c ON p.id_paises = c.id_paises AND e.ano = c.ano
LEFT JOIN populacao pop ON p.id_paises = pop.id_paises AND e.ano = pop.ano
GROUP BY p.id_paises, p.nome, cont.nome, e.ano;
```

# Risk Analysis

| Risk Category           | Description                                                                 | Mitigation Strategy                                                                 |
|------------------------|-----------------------------------------------------------------------------|-------------------------------------------------------------------------------------|
| Write Amplification    | Denormalized tables require recomputation of multiple source rows           | Limit refresh frequency; run during off-peak hours                                  |
| Lock Contention        | Large batch inserts may block concurrent reads                              | Use `CONCURRENTLY` index builds where possible; partitioned tables                  |
| Failure Modes          | ETL job failure can lead to stale data                                      | Retry logic + alerting on job failures                                              |
| Backfill Runtime       | Full rebuild may take minutes depending on volume                           | Schedule during low-load periods; incremental updates via CDC if available          |
| Storage Growth         | Denormalized tables increase storage footprint                              | Monitor growth; compress historical data if needed                                  |
