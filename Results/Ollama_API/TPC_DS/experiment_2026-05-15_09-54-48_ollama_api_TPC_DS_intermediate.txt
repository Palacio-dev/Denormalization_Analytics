# Denormalized Physical Schema for TPC-DS Read Workloads

## 1. Workload-Driven Analysis

### High-Cost Operations Identified:
1. **Multi-table joins** for order analysis (customer details + order headers + line items)
2. **Date-based filtering** requiring multiple date conversions
3. **Customer segmentation queries** needing joins across purchase types
4. **Inventory tracking** with warehouse-item-date lookups
5. **Return analysis** requiring cross-referencing with original orders

### Denormalization Candidates:
- Customer information duplicated in order tables
- Date dimensions pre-joined with fact tables
- Pre-aggregated sales summaries by customer/item/date
- Wide order tables combining header and line item data
- Inventory snapshots with item descriptions

## 2. Denormalization Design

### Key Denormalizations:
1. **Wide Order Tables**: Pre-joined customer, item, and date information
2. **Sales Summary Tables**: Daily/weekly/monthly aggregates by customer/item
3. **Inventory Snapshot Tables**: Current inventory levels with item details
4. **Return Analysis Views**: Combined return and original order data
5. **Covering Indexes**: Optimized for common filter patterns

-- ============================================================
--   Materialized Views for Analytics
-- ============================================================
CREATE MATERIALIZED VIEW mv_customer_segmentation AS
SELECT 
    customer_id,
    CASE 
        WHEN total_lifetime_value > 10000 THEN 'PREMIUM'
        WHEN total_lifetime_value > 5000 THEN 'GOLD'
        WHEN total_lifetime_value > 1000 THEN 'SILVER'
        ELSE 'BRONZE'
    END AS customer_segment,
    preferred_channel,
    avg_order_value
FROM d_customer_summary;

CREATE INDEX idx_mv_customer_segment 
    ON mv_customer_segmentation (customer_segment);

CREATE MATERIALIZED VIEW mv_item_performance AS
SELECT 
    item_id,
    item_name,
    item_category,
    SUM(total_quantity) AS total_units_sold,
    SUM(total_sales_amount) AS total_revenue,
    COUNT(*) AS transaction_days
FROM d_daily_sales_summary
WHERE summary_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY item_id, item_name, item_category;

CREATE INDEX idx_mv_item_performance_revenue 
    ON mv_item_performance (total_revenue DESC);

## Index Strategy Summary

1. **Composite indexes** aligned with common query filters (date + customer, item + date)
2. **Covering indexes** for frequently accessed columns in reporting queries
3. **Partitioning-ready design** with date-based clustering columns
4. **Selective denormalization** focused on eliminating the most expensive joins

This schema reduces query complexity by pre-joining related entities and pre-computing aggregates, significantly improving read performance for analytical workloads while maintaining data correctness through careful design.
