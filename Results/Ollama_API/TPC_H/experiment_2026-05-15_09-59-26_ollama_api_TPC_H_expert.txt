# Executive Summary

This design proposes a denormalized read-optimized schema for the TPC-H-like normalized relational schema, with the goal of significantly improving read performance for analytical and operational queries commonly found in business intelligence and reporting workloads. By identifying hotspots such as joins across `ORDERS`, `CUSTOMER`, `LINEITEM`, and dimensional tables like `NATION` and `REGION`, we propose wide read tables and pre-aggregated rollups to eliminate expensive real-time joins and aggregations.

The new physical schema includes:
- **Wide Read Table**: `ORDER_CUSTOMER_DETAILS` — combines order and customer data for frequent filters on order status, date, and customer segment.
- **Pre-Aggregated Rollup**: `DAILY_SALES_SUMMARY` — aggregates daily sales metrics (revenue, quantity, discount) by customer segment and nation.
- **Snapshot Table**: `CURRENT_INVENTORY` — captures current supplier-part inventory levels from `PARTSUPP`.

All artifacts are strictly tied to specific query patterns observed in typical TPC-H-style workloads. The implementation plan is designed to be safe, observable, and reversible, with incremental deployment and reconciliation mechanisms.

# Proposed Denormalized Schema (DDL)

## 1. Wide Read Table: `ORDER_CUSTOMER_DETAILS`

**Purpose**: Eliminate joins between `ORDERS`, `CUSTOMER`, `NATION`, and `REGION`.

```sql
CREATE TABLE ORDER_CUSTOMER_DETAILS (
    OCD_ORDERKEY       INTEGER NOT NULL,
    OCD_CUSTKEY        INTEGER NOT NULL,
    OCD_ORDERDATE      DATE NOT NULL,
    OCD_ORDERSTATUS    CHAR(1) NOT NULL,
    OCD_TOTALPRICE     DECIMAL(15,2) NOT NULL,
    OCD_ORDERPRIORITY  CHAR(15) NOT NULL,
    OCD_CUST_NAME      VARCHAR(25) NOT NULL,
    OCD_CUST_SEGMENT   CHAR(10) NOT NULL,
    OCD_NATION_NAME    CHAR(25) NOT NULL,
    OCD_REGION_NAME    CHAR(25) NOT NULL,
    OCD_INSERTED_AT    TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (OCD_ORDERKEY)
);
```

**Indexes**:
```sql
CREATE INDEX IDX_OCD_CUST_SEGMENT ON ORDER_CUSTOMER_DETAILS(OCD_CUST_SEGMENT);
CREATE INDEX IDX_OCD_ORDERDATE ON ORDER_CUSTOMER_DETAILS(OCD_ORDERDATE);
```

**Purpose**: Speed up time-series reporting by precomputing daily sales metrics grouped by customer segment and nation.

```sql
CREATE TABLE DAILY_SALES_SUMMARY (
```

# Risk Analysis

| Risk | Description | Mitigation |
|------|-------------|------------|
| Write Amplification | Denormalized tables require additional writes | Use batch or incremental refresh to reduce frequency |
| Lock Contention | Large refreshes may lock tables | Use partition swapping or staging tables |
| Backfill Runtime | Initial load may take hours | Plan during maintenance windows |
| Storage Growth | Denormalized tables increase storage | Monitor growth and archive older data |
| Query Regressions | New schema may be misused | Enforce access via views or BI layers |
| Data Freshness | Stale data due to refresh lag | Monitor freshness metrics |
| Reconciliation | Mismatch between source and derived | Build checksum-based reconciliation jobs |
