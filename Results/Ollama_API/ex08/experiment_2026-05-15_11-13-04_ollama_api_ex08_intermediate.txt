# Denormalized Physical Schema for Read Performance Optimization

## 1. Workload-Driven Analysis

### High-Cost Join Paths Identified:
- **Student Trajectory Analysis**: Requires joining aluno_ifb → matricula_ifb → curso_ifb → vinculo_empregaticio → remuneracao (5-table join)
- **School Performance Analysis**: Requires joining escola → desempenho_enem, escola → indicadores_escolares, escola → participacao_enem, escola → localizacao (4-table join)
- **Student Demographics with Outcomes**: Requires joining aluno_ifb → matricula_ifb → escola → localizacao (4-table join)

### Denormalization Candidates:
1. Student trajectory data (frequent multi-table joins)
2. School performance metrics (pre-aggregated indicators)
3. Student employment outcomes with school data

## 2. Denormalization Design

### Proposed Denormalizations:
1. **Wide Fact Table**: fato_aluno_trajetoria combining student demographics, enrollment, course, and employment data
2. **Precomputed Aggregate Table**: dim_escola_indicadores_anuais combining school metrics with performance data
3. **Covering Indexes**: For common query patterns
4. **Partitioning**: By year for temporal queries

## Key Optimizations Implemented:

1. **Eliminated Multi-Table Joins**: The main fact table removes joins between 5+ tables for student trajectory analysis
2. **Precomputed Aggregates**: School performance metrics are pre-aggregated by year
3. **Partitioning**: Main fact table partitioned by year for efficient temporal queries
4. **Covering Indexes**: Designed specifically for common query patterns
5. **Materialized Views**: For expensive aggregations that don't need real-time updates
6. **Summary Tables**: For regional performance metrics to avoid runtime calculations
7. **Optimized Data Types**: Used appropriate numeric types and removed unnecessary serial columns in denormalized tables

This schema design reduces query complexity from multiple joins to single table scans or simple index lookups, significantly improving read performance for the identified workload patterns.
