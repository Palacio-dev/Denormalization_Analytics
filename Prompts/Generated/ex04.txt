EXPERT:
You are a Principal Data Engineer and Data Warehouse Architect tasked with redesigning a transactional relational model into an analytics-optimized physical model.

Objective:

Transform the provided normalized OLTP schema into a denormalized analytical schema optimized for high-performance read workloads, dashboarding, ad hoc analysis, and large-scale aggregation queries.

Assumptions:

The resulting model will serve analytical and decision-support use cases.
Read latency and query simplicity are prioritized over update efficiency.
Data is loaded through scheduled ETL/ELT pipelines.
Controlled redundancy is acceptable.
Historical analysis requirements should be considered where relevant.

Instructions:

Perform the following steps:

1. Workload-Oriented Assessment

Analyze the schema and infer likely analytical access patterns, including:

Common business entities
High-cardinality dimensions
Frequently joined tables
Potential aggregation paths
Candidate measures and metrics
2. Analytical Modeling Strategy

Determine the most appropriate target architecture, choosing among:

Star schema
Snowflake-to-star conversion
Wide denormalized tables
Data marts
Hybrid dimensional model

Justify the chosen architecture based on expected analytical workloads.

3. Denormalization Design

For each relationship:

Evaluate join elimination opportunities.
Identify attributes that should be embedded into fact-like structures.
Remove unnecessary normalization layers.
Preserve lineage through business keys and surrogate keys where appropriate.
Recommend handling of slowly changing attributes if relevant.
4. Physical Schema Generation

Produce a complete physical model including:

Table definitions
Column definitions
Primary keys
Foreign keys (if retained)
Partitioning recommendations
Clustering/sort key recommendations
Indexing recommendations where applicable
5. Performance Review

For each design decision, explain:

Expected reduction in join complexity
Impact on scan efficiency
Impact on storage consumption
ETL maintenance implications
Scalability considerations
6. Final Deliverables

Provide:

Current schema assessment.
Target analytical architecture.
Denormalization rationale.
Complete physical schema (DDL).
Data lineage considerations.
Performance and maintenance trade-off analysis.
Additional recommendations for future analytical growth.

Input Schema:

[INSERT RELATIONAL PHYSICAL MODEL HERE]

Generate a production-ready analytical physical model and explicitly state all assumptions made during the redesign process.
