Act as a database architect. I need to transform an existing normalized relational physical model into a denormalized
physical schema for analytics and read-heavy workloads.


Your task:
- Redesign the schema for analytics consumption.
- Reduce joins where possible by denormalizing strategically.
- Include table structures, key columns, duplicated dimensions, summary tables, and indexing or partitioning recommendations.
- Explain the mapping from the normalized source to the denormalized target.
- Preserve data consistency where needed, but prioritize read performance.
- Produce a new physical schema that could support a data mart or reporting layer.
- State the tradeoffs in query speed, storage, and data maintenance.

Deliverables:
    Provide your answer in three distinct sections:
        Section 1: Explain structural decisions (merges, splits, table creation), attribute-level changes (added redundancies,
                derived columns, removals), and any tradeoffs acknowledged
        Section 2: For each denormalization decision, provide the query pattern that motivated it. If no workload context 
                was provided, provide the inferred query.
        Section 3: A SQL file containing the complete DDL creation script of the denormalized model.
