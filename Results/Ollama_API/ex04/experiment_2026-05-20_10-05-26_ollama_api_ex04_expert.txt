# **Executive Summary**

This document outlines a denormalized read-optimized schema designed to significantly improve query performance on analytical workloads involving poverty metrics, socio-economic indicators, and country-level development statistics. The approach focuses on reducing join overhead by pre-joining frequently accessed dimension tables (`country`, `region`) into fact tables such as `survey` and related demographic/health/economic entities. Additionally, we introduce snapshot tables for point-in-time analysis of current country status and aggregate rollups grouped by geographic region or temporal buckets.

All transformations preserve logical correctness via strict lineage tracking and controlled refresh strategies with defined staleness windows (< 1 hour). The implementation is designed to be fully reversible using materialized views or scheduled ETL pipelines without reliance on triggers.

### 4. Decile Summary View: `v_decile_distribution`

**Purpose:** Simplify access to decile distributions with associated survey metadata.

```sql
CREATE MATERIALIZED VIEW IF NOT EXISTS public.v_decile_distribution AS
SELECT 
    d.id_decile,
    s.country_code,
    c.country_name,
    r.region_name,
    s.survey_year,
    d.name AS decile_label,
    d.value AS income_share
FROM decile d
JOIN survey s ON d.id_survey = s.id_survey
JOIN country c ON s.country_code = c.country_code
JOIN region r ON c.region_code = r.region_code;
```

**Refresh Strategy:** Refresh nightly after main ETL pipeline completes.

## **Risk Analysis**

### Write Amplification
Denormalized writes increase due to cascading updates. Mitigated via:
- Batched refreshes instead of real-time propagation
- Logical partitioning where applicable

### Lock Contention
High write volume may block readers temporarily. Solution:
- Use MVCC-compatible engines (PostgreSQL, Snowflake)
- Employ table swapping techniques for atomic refreshes

### Failure Modes
- Data drift between base and summary tables → Monitor via reconciliation diffs
- Long-running refresh jobs → Implement timeout alerts and partial rebuilds

### Backfill Runtime
Estimated full backfill time:
- `country_survey_summary`: ~30 mins
- `current_country_status`: ~10 mins
- `regional_annual_stats`: ~5 mins
- `v_decile_distribution`: ~5 mins

### Storage Growth
