You are a senior data architect and physical database designer. I need you to redesign a normalized relational OLTP 
schema into a denormalized analytics-ready physical model.

Objective:
I will provide a normalized relational physical schema from a transactional system, and I need you to redesign it into 
a denormalized analytics-oriented physical schema suitable for a data mart or reporting layer. The objective is to improve 
read performance and simplify analytical queries by reshaping the base into an analytics-ready version.

What you must do:

- Reverse-engineer the source schema and infer business processes, entities, and relationships.
- Define the analytical grain explicitly for each target table.
- Choose an appropriate denormalization pattern:
    - star schema
    - snowflake with partial denormalization
    - wide analytical table
    - selective attribute duplication
    - hybrid model
- Justify the chosen pattern based on workload and query shapes.
- Model conformed dimensions where relevant.
- Flatten or pre-join descriptive attributes that are repeatedly used in analytics.
- Propose surrogate keys, natural key retention, and any bridge tables if many-to-many relationships exist.
- Recommend partitioning, indexing, clustering, materialized views, or other physical optimizations appropriate for the target DBMS.
- Identify potential data quality risks introduced by denormalization.
- Provide migration mapping from source tables to target structures.
- Explain what should be materialized now versus computed at query time.

Required output:
    Provide your answer in three distinct sections:
        Section 1: Explain structural decisions (merges, splits, table creation), attribute-level changes (added redundancies,
                derived columns, removals), and any tradeoffs acknowledged
        Section 2: For each denormalization decision, provide the query pattern that motivated it. If no workload context 
                was provided, provide the inferred query.
        Section 3: A SQL file containing the complete DDL creation script of the denormalized model.
