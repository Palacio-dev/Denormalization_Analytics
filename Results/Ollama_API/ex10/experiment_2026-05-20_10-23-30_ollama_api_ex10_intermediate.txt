# Denormalized Schema for Read Performance Optimization

## 1. Workload-Driven Analysis

### High-Cost Join Paths Identified:
1. **Country Information Lookups**: Queries frequently join `paises` with `continente` to get continent names
2. **Energy Consumption Analysis**: Queries join `consumo_anual` with `paises`, `continente`, and `fonte_energia` 
3. **Emissions Analysis**: Queries join `emissao_g_efeito_estufa` with `paises` and `continente`
4. **Population Analysis**: Queries join `populacao` with `paises` and `continente`
5. **Cross-Domain Analysis**: Queries frequently join multiple fact tables (`consumo_anual`, `emissao_g_efeito_estufa`, `populacao`) through `paises`

### Denormalization Candidates:
- Pre-join country and continent information
- Include energy type names directly in consumption data
- Create wide fact tables combining related metrics
- Add summary tables for common aggregations

## 2. Denormalization Design

### Proposed Denormalizations:
1. **Wide Fact Table**: `fato_energia_pais_ano` combining consumption, emissions, and population
2. **Precomputed Aggregates**: `agregado_consumo_continente_ano` for continent-level consumption summaries
3. **Covering Indexes**: Optimized indexes for common query patterns
4. **Redundant Attributes**: Continent name in country dimension to avoid joins

CREATE TABLE IF NOT EXISTS public.agregado_emissao_continente_ano (

## Query Pattern Alignment

### 1. Country Information Queries
**Before**: Join `paises` → `continente`  
**After**: Direct access in `dim_pais` table

### 2. Energy Consumption Analysis
**Before**: Join `consumo_anual` → `paises` → `continente` → `fonte_energia`  
**After**: Direct access in `fato_energia_pais_ano` with all information pre-joined

### 3. Cross-Domain Analysis (Consumption + Emissions + Population)
**Before**: Multiple joins across 5 tables  
**After**: Single table access with `fato_energia_pais_ano`

### 4. Continent-Level Summaries
**Before**: Aggregation with multiple joins  
**After**: Precomputed in `agregado_consumo_continente_ano` and `agregado_emissao_continente_ano`

### 5. Time-Series Analysis
**Before**: Filtering through multiple tables  
**After**: Optimized with partitioned tables and time-based indexes

This denormalized schema reduces query complexity from 4-5 table joins to single table access for most common queries, significantly improving read performance while maintaining data correctness through careful design.
