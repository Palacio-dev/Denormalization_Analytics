## Denormalization Techniques Applied:

### 1. **Collapsing Relations (Technique 3)**
- Combined 9 tables into the main fact table: `aluno_ifb`, `matricula_ifb`, `curso_ifb`, `escola`, `localizacao`, `indicadores_escolares`, `desempenho_enem`, `participacao_enem`, `vinculo_empregaticio`, `remuneracao`
- Reduced complex join paths to single table access

### 2. **Adding Redundant Attributes (Technique 1)**
- School names, locations, and performance metrics stored directly with student records
- Course information embedded with enrollment data
- Employment details stored with student demographics

### 3. **Adding Derived Attributes (Technique 2)**
- `tempo_ate_empregabilidade`: Days from graduation to first job
- `salario_vs_carga_horaria`: Hourly wage calculation
- `performance_geral_enem`: Average ENEM scores
- `ranking_escola_estado`: Comparative performance metrics

### 4. **Vertical Partitioning Consideration (Technique 4)**
- Separated school analytics into `dim_escola_analitica` for focused queries
- Created `dim_curso_analitica` for course-specific analysis
- Maintained time dimension for proper temporal analysis

3. **Supports Aggregation**: Pre-computed metrics enable fast reporting and dashboard generation
