EXPERT:
You are a senior data architect specializing in physical data modeling for analytics. I need a denormalized physical 
schema derived from a normalized relational source system, with the primary goal of optimizing read performance for analytical workloads.

Design the target model with rigor:
- Analyze the source schema, entity relationships, and workload patterns.
- Define the analytical grain explicitly.
- Recommend a denormalized physical design appropriate for the access patterns: wide tables, star schema, snowflake reduction,
  summary tables, materialized views, or hybrid patterns where justified.
- Minimize joins for frequent reads while preserving analytical correctness.
- Specify surrogate keys, conformed dimensions, degenerate dimensions, slowly changing dimension strategy if relevant, 
  and aggregation strategy.
- Include physical storage considerations such as partitioning, clustering, indexing, compression, and refresh strategy.
- Provide SQL DDL for the denormalized target schema.
- Provide an implementation approach for loading from the operational source into the analytics model.
- Call out edge cases, semantic tradeoffs, and data quality risks.
- Optimize for BI/reporting latency rather than write efficiency.

- Proposed physical model

- Source-to-target mapping
- Load/refresh strategy
- Performance rationale
- Tradeoff analysis
