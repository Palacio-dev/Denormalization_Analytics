## Key Optimizations Implemented:

### 1. **Wide Fact Tables**
- `fact_escola_performance`: Pre-joined school data with IDEB/SAEB metrics
- `fact_curso_performance`: Pre-joined higher education data with ENADE metrics

### 2. **Aggregate Tables**
- `agg_uf_performance`: State-level summaries for quick reporting
- `agg_municipio_performance`: Municipality-level summaries

### 3. **Materialized Views**
- `mv_escola_ranking_uf`: Pre-computed rankings by state
- `mv_curso_ranking_uf`: Pre-computed higher education rankings

### 4. **Indexing Strategy**
- **Composite indexes** aligned with common query filters (UF, ano, performance metrics)
- **Covering indexes** for frequently accessed columns
- **Partitioning** by year for time-series queries

### 5. **Dimension Tables**
- `dim_uf` and `dim_municipio` for geographic lookups
- Maintained for referential integrity while avoiding joins in fact tables

### 6. **Partitioning**
- Year-based partitioning on fact tables for efficient time-series queries
- Automatic partition management for new years

This design eliminates the need for complex joins in read-heavy workloads while maintaining data correctness through careful denormalization. The schema supports common analytical queries with minimal I/O operations.
