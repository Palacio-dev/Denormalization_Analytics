1. Creating a Escola table

Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.

The attibutes from the following table were united in a single table: TipoCapital, Estado, Municipio, Escola,
TipoLocalizacao, TipoRede, NotaSAEB, NotaIDEB, IndicadoresAlunos, PorcentagemClassificacaoAlunos, TaxasAprovacao and 
AprovacaoSerie .This way, the one-to-may relationships are collapsed, and redundant attributes
that would just get searched from a JOIN query are now on the same big table.

2. Creating a ClassificacaoSE table

Techniques Used: 3 - Collapsing Relations and 1 - Adding Redundant Attributes.

The table ClassificacaoSocioEconomica and the table NivelEscolaridade turned into a big table with a one-to-many 
relationship with the Escola table. 


It is important to notice that the tables are connected with the "idClassificacaoSE" attribute, which is a relationship
one-to-many, basically, one "escola" has many "idClassificacaoSE", because it has many students. This way,it is possible
to get all the information previously possible on the normalized model.
