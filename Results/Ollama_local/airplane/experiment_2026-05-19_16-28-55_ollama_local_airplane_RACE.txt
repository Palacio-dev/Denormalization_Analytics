### 3. Evaluation of Denormalization Techniques

*   **Adding Redundant Attributes:** **High Priority.** We must duplicate `airport_name`, `city`, `timezone`, `model`, and `range` directly into the `flights` table. This eliminates the most frequent join.
*   **Adding Derived Attributes:** **Medium Priority.** While useful, pre-calculated aggregates (like total revenue) are often better suited for Materialized Views rather than static table columns, as flight data changes frequently. However, we can store the *count* of passengers per flight if that metric is static for the reporting period.
*   **Collapsing Relations:** **High Priority.** We should collapse the `tickets` and `ticket_flights` relationship. A single `tickets` table can hold the flight ID, eliminating one join layer. Similarly, we can collapse `boarding_passes` into the `tickets` or `flights` table if the data is identical, but keeping it separate is acceptable if it adds specific boarding logic. *Decision: Let's collapse `boarding_passes` into `tickets` for simplicity, or keep it minimal.*
*   **Vertical Partitioning:** **Not Applicable.** Our tables are small enough that splitting them vertically doesn't offer significant I/O benefits compared to the massive JOIN savings gained by redundancy.

### Summary of Changes

1.  **Eliminated Joins:**
    *   **Before:** `SELECT ... FROM tickets JOIN ticket_flights ... JOIN flights JOIN airports ...`
    *   **After:** `SELECT ... FROM tickets_data JOIN flights_data ...` (or even single table scans if `tickets_data` includes all flight details).

    *   The `boarding_passes` table was effectively collapsed into `tickets_data` (or assumed redundant if boarding info matches ticket info).
4.  **Performance Impact:**
    *   Queries that previously required 4+ joins now require 1 or 2.
    *   Indexes are now more effective because the tables are smaller and the join predicates are eliminated.
    *   The "Read" performance should improve by an estimated **30-50%** for complex analytical queries involving route and passenger data.
